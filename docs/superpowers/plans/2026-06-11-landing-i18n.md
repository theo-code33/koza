# kōza — i18n de la landing page — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Faire passer la landing page publique (9 composants + `/`, `/privacy`, `/terms`) sous next-intl, FR + EN complets, en réutilisant l'infrastructure i18n existante.

**Architecture:** `useTranslations` (API isomorphe synchrone) dans tous les composants/pages landing — pas de `getTranslations` (garde les composants serveur non-async donc testables). Listes (features/benefits/faq) via `t.raw(...)`. Markup inline (benefits) via `t.rich(...)`. Métadonnées des pages légales via `generateMetadata` + `getTranslations`. Pas de switcher : la landing suit `resolveLocale()` (cookie → DB → `fr`).

**Tech Stack:** Next 16 App Router, React 19, next-intl 4, Vitest + Testing Library.

---

## Conventions pour chaque commit

- Branche : **`feat/landing-i18n`** (déjà créée depuis `main`). Jamais de commit sur `main`.
- Conventional Commits, anglais, ≤72 car., scope `i18n` (ou `landing`).
- Aucun trailer `Co-authored-by`. Merge commit manuel par le mainteneur.
- Avant chaque commit : `npm run format`, `npm run lint` (0 erreur ; warnings RHF pré-existants tolérés), `npm run test` au vert.
- Code touchant les types : `DATABASE_URL="postgresql://build:build@localhost:5432/build?schema=public" npm run build` avant de committer.

## Rappels next-intl (déjà en place)

- Catalogues : `src/locales/fr.json` (source) et `src/locales/en.json`, namespacés. Le `NextIntlClientProvider` est dans `src/app/layout.tsx`. Test de parité des clés : `src/locales/parity.test.ts`.
- Client **et** serveur sync : `const t = useTranslations("landing");` puis `t("hero.title")`.
- Liste brute : `const items = t.raw("landing.features.items") as { title: string; description: string }[];` puis `items.map(...)`.
- Markup inline : `t.rich("landing.benefits.list1", { b: (chunks) => <strong>{chunks}</strong> })`.
- Helper de test : `renderWithIntl` (`src/test/render-with-intl.tsx`) injecte le `fr.json` réel.

## File Structure

- `src/locales/fr.json`, `src/locales/en.json` (modify) — namespaces `landing` + `legal`.
- `src/components/landing/Navbar.tsx`, `HeroEnhanced.tsx`, `Hero.tsx`, `Features.tsx`, `Benefits.tsx`, `FAQ.tsx`, `CTAFinal.tsx`, `Footer.tsx`, `JoselineEasterEgg.tsx` (modify).
- `src/app/privacy/page.tsx`, `src/app/terms/page.tsx` (modify).
